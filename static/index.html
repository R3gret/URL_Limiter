Centralized Rate Limiter

High-performance Standalone Microservice

Overview

This is a centralized Rate Limiting Microservice built in Go. It allows all of your web projects (frontends, backends, APIs) to share a single, lightning-fast Token Bucket rate limiter, tracking usage via any identifier (like a User ID or IP).

How to use it with other projects

Simply make an HTTP request to this microservice before fulfilling a user's request in your own project.

  1. Your backend receives a request from a user.
  2. Your backend pauses to send a POST /api/check to this service.
  3. If the microservice returns allowed: true, your backend proceeds. If false, your backend blocks the user!
// In your Node.js/Next.js/Express backend
const checkRateLimit = async (userId) => {
  const response = await fetch('http://localhost:8080/api/check', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ identifier: userId })
  });
  
  const data = await response.json();
  if (!data.allowed) {
    throw new Error("Rate limit exceeded!");
  }
  // Proceed with fulfilling the request
};
# In your Python (Django/FastAPI) backend
import requests

def check_rate_limit(user_id):
    response = requests.post(
        'http://localhost:8080/api/check',
        json={'identifier': user_id}
    )
    
    data = response.json()
    if not data.get('allowed'):
        raise Exception("Rate limit exceeded!")
    # Proceed with fulfilling the request
